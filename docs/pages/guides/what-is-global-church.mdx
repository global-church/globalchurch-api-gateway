# What is Global.Church?

This page will cover:
- The data silo problem in global missions — no single organization can see the full picture of who is engaged where
- How Global.Church solves it with a shared ontology (GC Ontology) and a unified knowledge graph (GraphDB)
- The ecosystem components: platform (data management), engage (engagement dashboard), API gateway, and GraphDB triplestore
- Who it's for: mission agencies, church networks, denominations, and developers building on missions data
- How the two integration patterns work: batch bridge materialization and runtime SPARQL federation

> Content will be provided by the documentation content authors.
