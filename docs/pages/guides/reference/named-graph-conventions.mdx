# Named Graph Conventions

This page will cover:
- The named graph model: each data source lives in its own named graph for isolation and provenance
- Standard graph URIs: `https://data.global.church/joshua-project/`, `https://data.global.church/imb/`, `https://data.global.church/his-registries/`, and others
- The default graph: where the GC Ontology itself is stored
- Production vocabularies graph: `https://data.global.church/vocabs/` for OrganizationType, BeliefType, Denomination schemes
- Resource graphs: `https://data.global.church/imb/resources/`, `https://data.global.church/jfilm/resources/`, `https://data.global.church/fcbh/resources/`
- How to scope SPARQL queries to specific graphs using `GRAPH` clauses
