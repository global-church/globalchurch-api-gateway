# Contributing Linked Data

This page will cover:
- Using `POST /v0/ingest` to submit RDF triples (Turtle, N-Triples, or RDF/XML format)
- Namespace and URI conventions your data must follow (ontology.global.church prefixes, data.global.church URIs)
- Named graph isolation: how contributed data is kept separate from other sources
- SHACL validation: what shapes your data must conform to before acceptance
- The review workflow: how data stewards approve or reject contributed data
- Building a bridge: the pattern for creating a batch pipeline from your data source to GraphDB

> Content will be provided by the documentation content authors.
