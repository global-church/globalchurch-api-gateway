# Data Modeling Cookbook

This page will cover:
- How to model an Organization with contact, classification, and geographic properties
- Representing People Group assessments using the 3-block pattern (PeopleGroup + Assessment + AssessmentResult)
- Linking resources to people groups using `gc:MissionResource` with orthogonal classification dimensions (domain, format, function)
- Using SKOS concept schemes for standardized vocabularies (religions, languages, denominations)
- Provenance tracking with PROV-O: who contributed what data and when
- Common anti-patterns to avoid: flat properties vs. structured assessments, duplicating vs. linking

> Content will be provided by the documentation content authors.
