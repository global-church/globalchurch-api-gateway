# Reviewing Claims

This page will cover:
- What claims are: organization claims (linking a user to an org) and engagement claims (reporting field activity)
- The review workflow: submitted → pending review → approved/rejected by admin or support role
- Using the admin claims dashboard at platform.global.church/admin/claims
- API routes involved: `GET /v0/admin/claims`, `POST /v0/admin/claims/review`, `GET /v0/engagement-claims`
- RBAC requirements: which roles (admin, support) can review which claim types
- Best practices for verifying claim accuracy before approval

> Content will be provided by the documentation content authors.
